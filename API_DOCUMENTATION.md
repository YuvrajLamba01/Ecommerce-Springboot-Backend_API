# E-Commerce Backend API Documentation

## Base URL
```
http://localhost:8080/api
```

## API Endpoints

### 1. Product Management

#### Get All Products
- **Method**: `GET`
- **URL**: `/api/products`
- **Description**: List all products

#### Get Product by ID
- **Method**: `GET`
- **URL**: `/api/products/{id}`
- **Description**: Get a single product by ID

#### Search Products
- **Method**: `GET`
- **URL**: `/api/products/search?name=laptop`
- **Description**: Search products by name

#### Get Products by Category
- **Method**: `GET`
- **URL**: `/api/products/category/{category}`
- **Description**: Filter products by category

#### Create Product
- **Method**: `POST`
- **URL**: `/api/products`
- **Request Body**:
```json
{
  "name": "Laptop",
  "description": "High-performance laptop",
  "price": 999.99,
  "stockQuantity": 50,
  "category": "Electronics",
  "imageUrl": "https://example.com/laptop.jpg"
}
```

#### Update Product
- **Method**: `PUT`
- **URL**: `/api/products/{id}`
- **Request Body**: Same as Create Product

#### Delete Product
- **Method**: `DELETE`
- **URL**: `/api/products/{id}`

---

### 2. Shopping Cart Management

#### Get Cart Items
- **Method**: `GET`
- **URL**: `/api/cart/{userId}`
- **Description**: Get all items in user's cart

#### Add to Cart
- **Method**: `POST`
- **URL**: `/api/cart`
- **Request Body**:
```json
{
  "userId": "user123",
  "productId": "prod456",
  "quantity": 2
}
```

#### Remove from Cart
- **Method**: `DELETE`
- **URL**: `/api/cart/{cartItemId}`

#### Clear Cart
- **Method**: `DELETE`
- **URL**: `/api/cart/clear/{userId}`

#### Get Cart Total
- **Method**: `GET`
- **URL**: `/api/cart/{userId}/total`

---

### 3. Order Management

#### Get All Orders
- **Method**: `GET`
- **URL**: `/api/orders`

#### Get Order by ID
- **Method**: `GET`
- **URL**: `/api/orders/{id}`

#### Get Orders by User
- **Method**: `GET`
- **URL**: `/api/orders/user/{userId}`

#### Create Order
- **Method**: `POST`
- **URL**: `/api/orders`
- **Request Body**:
```json
{
  "userId": "user123",
  "shippingAddress": "123 Main St, City, State 12345"
}
```

#### Update Order Status
- **Method**: `PATCH`
- **URL**: `/api/orders/{orderId}/status?status=CONFIRMED`
- **Query Params**: 
  - `status`: PENDING, CONFIRMED, SHIPPED, DELIVERED, CANCELLED

#### Cancel Order
- **Method**: `DELETE`
- **URL**: `/api/orders/{orderId}`

---

### 4. Payment Processing

#### Initiate Payment
- **Method**: `POST`
- **URL**: `/api/payments`
- **Request Body**:
```json
{
  "orderId": "order123",
  "userId": "user123",
  "amount": 1999.99,
  "paymentMethod": "CREDIT_CARD"
}
```
- **Payment Methods**: CREDIT_CARD, DEBIT_CARD, PAYPAL, UPI

#### Get Payment by ID
- **Method**: `GET`
- **URL**: `/api/payments/{id}`

#### Get Payment by Order ID
- **Method**: `GET`
- **URL**: `/api/payments/order/{orderId}`

---

### 5. Payment Webhook (Mock Service)

#### Payment Webhook Handler
- **Method**: `POST`
- **URL**: `/api/webhooks/payment`
- **Request Body**:
```json
{
  "paymentId": "pay123",
  "transactionId": "txn456",
  "status": "COMPLETED",
  "message": "Payment successful"
}
```
- **Status Values**: PENDING, COMPLETED, FAILED, REFUNDED

## Testing Workflow

### Complete E-Commerce Flow:

1. **Create Products**
   ```
   POST /api/products
   ```

2. **List Products**
   ```
   GET /api/products
   ```

3. **Add Products to Cart**
   ```
   POST /api/cart
   Body: {"userId": "user1", "productId": "prod1", "quantity": 2}
   ```

4. **View Cart**
   ```
   GET /api/cart/user1
   ```

5. **Get Cart Total**
   ```
   GET /api/cart/user1/total
   ```

6. **Create Order (Places order from cart)**
   ```
   POST /api/orders
   Body: {"userId": "user1", "shippingAddress": "123 Main St"}
   ```
   Note: This automatically clears the cart

7. **Initiate Payment**
   ```
   POST /api/payments
   Body: {"orderId": "order_id_from_step6", "userId": "user1", "amount": 1999.99, "paymentMethod": "CREDIT_CARD"}
   ```

8. **Simulate Payment Webhook (Mock)**
   ```
   POST /api/webhooks/payment
   Body: {"paymentId": "payment_id_from_step7", "transactionId": "TXN123", "status": "COMPLETED", "message": "Success"}
   ```

9. **Verify Order Status Updated**
   ```
   GET /api/orders/{orderId}
   ```
   Status should be "CONFIRMED"

---

## Status Codes

- `200 OK` - Success
- `201 Created` - Resource created
- `204 No Content` - Successful deletion
- `404 Not Found` - Resource not found
- `500 Internal Server Error` - Server error

---

## Notes

- MongoDB must be running on `localhost:27017`
- All IDs are auto-generated by MongoDB
- Cart is automatically cleared after order creation
- Order status is updated automatically when payment is completed
- Payment processing uses a mock service (returns dummy transaction ID)
